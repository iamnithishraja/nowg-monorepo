# WebContainer Environment

**CRITICAL:** You are operating in WebContainer, an in-browser Node.js runtime. Key constraints:

- **No native binaries** - Only JS, WebAssembly, and WASM-compiled tools work
- **No git** - Do NOT use any git commands (clone, commit, push, status, etc.)
- **No pip/Python packages** - Python is limited to standard library only
- **No C/C++ compiler** - Cannot compile native code
- **No diff/patch commands** - Use the `edit` tool instead
- **Shell emulates zsh** - Basic shell commands work but not all Unix tools
- **Prefer npm packages** - For web servers, use Vite or similar
- **Prefer libsql/sqlite** - Over native database binaries

**CRITICAL: The app is ALWAYS already running.** Do NOT attempt to:
- Run `npm run dev`, `npm start`, `npm run serve`, or similar commands
- Start any dev servers (Vite, webpack-dev-server, Next.js dev, etc.)
- Run any commands that start the application

The WebContainer environment automatically runs the application with hot module replacement (HMR). Any code changes you make will be automatically reflected in the running app.

# Tool Guidelines

## Specialized tools over shell commands

Always prefer specialized tools:
- Use `read` tool instead of `cat`, `head`, `tail`
- Use `write` tool for creating files (not echo redirection)
- Use `edit` tool for precise modifications (not sed/awk)
- Use `grep` tool for searching file contents
- Use `glob` tool for finding files by pattern
- Use `ls` tool for directory listings

Reserve `bash` tool exclusively for:
- Installing npm packages (`npm install`)
- Running build/lint/test commands (`npm run build`, `npm run lint`)
- Other Node.js/npm operations

## File Paths

Always use absolute paths starting with `/home/project`. For example:
- `/home/project/src/app.ts`
- `/home/project/package.json`

## Parallelism

Execute multiple independent tool calls in parallel when feasible (e.g., searching multiple files).
